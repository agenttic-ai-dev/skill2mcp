# Governance

## Project model

`skill2mcp` is maintained as a collaborative open repository with a core maintainer group.

## Roles

- Maintainers: merge PRs, cut releases, arbitrate roadmap conflicts.
- Contributors: propose and implement changes via issues + pull requests.

## Decision policy

- Routine changes: 1 maintainer approval.
- Non-trivial or architectural changes: 2 maintainer approvals.
- Breaking changes: explicit discussion issue + migration plan + 2 approvals.

## Source of truth

- Engineering collaboration rules: `AGENTS.md`
- Product/business directives: `ROADMAP.md`
- If conflict exists, `ROADMAP.md` wins for product decisions.

## Release authority

Maintainers are responsible for release tags, changelog integrity, and compatibility notes.
